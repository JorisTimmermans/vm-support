How to create a MULTIPLY VM? (draft)

The MULTIscale SENTINEL land surface information retrieval Platform (MULTIPLY) provides a VM with Jupyter notebooks and data processors you can work with. You need a Mundi account to launch your personal MULTIPLY VM on Mundi and use it from the browser of your local machine. On Mundi your VM accesses available Sentinel data directly avoiding download.

1. Create your Mundi account

You need to register with Mundi and sign a contract to become user of the cloud infrastructure of Mundi with access to Sentinel data. The cloud provider of Mundi is OTC. The use of cloud resources is linked to costs. https://mundiwebservices.com/
https://mundiwebservices.com/offer
https://mundiwebservices.com/contact

2. Create your Multiply VM

The Multiply VM (instance) is based on a prepared Multiply Image (template). Depending on the Mundi policy you may either find the image among the public images, or you can download the image file from https://multiply.obs-website.eu-de.otc.t-systems.com/images/multiply1-20191215.vdi (9 GB) and create an image from it in your account. The VM is created from the image using the following steps:
  • Login into your account at https://auth.otc.t-systems.com/authui/login.action?service=https://console.otc.t-systems.com/ecm/#/login .
  • Select Elastic Cloud Server (ECS), Create ECS (in the upper right), select flavor s2.large.4 (2 cores, 8 GB), select image multiply1-20191215 with a system disk of 30 GB, add a data disk of 200 GB.
  • Select your default network and let the OTC choose the internal IP, use the default security group initially (you may modify this and create a more restricted one with access only from your local network), and assign an external IP (EIP).
  • Set the ECS name to e.g. mymultiplyvm, and create a key pair (unless you have one already) and select it, store the private key on your local machine, and protect the file against reading by anyone except your local user.
  • Confirm, wait for the VM to be created
If the image is not offered as a public image you can do the following to create your private image:
  • Create a simple Ubuntu VM on the OTC with at least 10 GB free disk space.
  • Install s3cmd (sudo apt install s3cmd). Use s3cmd --configure to set up your account for S3 access.
  • Copy the Multiply image file with s3cmd get s3://multiply/images/multiply1-20191215.vdi
  • Create your own bucket and copy the image file into it with s3cmd mb s3://mybucketname; s3cmd put multiply1-20191215.vdi s3://mybucketname/images/multiply1-20191215.vdi
  • Use the Image Management service to create your private image from the file. Select IMS, create image, select source 'image file', select the .vdi file in your bucket, make sure you check 'Enable automatic configuration', set the system disk to 30 GB, set name to multiply1-currentdate, press 'create now'

3. Configure and use the VM

  • Login at the VM from your local machine with ssh -i multiply@ , e.g. ssh -i multiply_key.priv multiply@80.158.17.186 .
  • Start the Jupyter notebook server with nohup jupyter-lab ~/software/multiply-ui/notebooks > jupyter-lab.out &
  • Start the ... service with ...
  • Access http://:8888/ with your local browser
  • Find more information at https://multiply.readthedocs.io .
Do not forget to stop and maybe delete the VM when it is not used to save costs.
